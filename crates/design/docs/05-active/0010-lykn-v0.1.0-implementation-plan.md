# Lykn v0.1.0 — Implementation Plan

**Generated from**: DD-01 through DD-09, plus post-DD review notes
**Date**: 2026-03-24
**Goal**: "Can write a real program" — HTTP server, CLI tool, utility module

---

## Architecture Reminder

```
.lykn source → reader.js (s-expr parser) → compiler.js (s-expr → ESTree AST) → astring (ESTree → JS)
```

The reader is **unchanged** for v0.1.0. All work is in `src/compiler.js`.
The Rust reader/formatter also needs no changes.

## Phase 1 — Foundation

**Dependencies**: None. Must be done first.
**Estimated effort**: ~1 day

### 1.1 camelCase Conversion (DD-01)

Implement `toCamelCase()` as a character-walk (not regex).

| Input | Output | Rule |
|-------|--------|------|
| `my-function` | `myFunction` | Standard: hyphen + letter → uppercase |
| `get-x` | `getX` | Single-letter segment |
| `get-HTTP-response` | `getHTTPResponse` | Uppercase preserved |
| `-foo` | `_foo` | Leading hyphen → underscore |
| `--foo` | `__foo` | Multiple leading → same count underscores |
| `foo-` | `foo_` | Trailing hyphen → underscore |
| `foo--bar` | `fooBar` | Consecutive mid-word → single boundary |
| `JSON` | `JSON` | No hyphens → no conversion |
| `_private` | `_private` | Existing underscores pass through |

Apply to every `Identifier` name. Each segment of a colon-split atom
is converted independently.

### 1.2 Colon Splitting (DD-01)

In `compileExpr`, when processing an `atom` node:

1. Check if `node.value` contains `:` and does NOT start with `:`
2. Split on `:`
3. Build chained `MemberExpression` (computed: false) from left to right
4. Apply `toCamelCase` to each segment independently
5. If first segment is `this` → emit `ThisExpression`
6. If first segment is `super` → emit `Super`

**Errors**:
- Leading `:` → "reserved for future use"
- Trailing `:` (e.g., `foo:`) → compile-time error
- Bare `:` → compile-time error
- Numeric segment (e.g., `obj:0`) → error, use `(get obj 0)`

### 1.3 `get` Form — Computed Access (DD-01)

New built-in form: `(get obj key)` → `MemberExpression` with
`computed: true`. Exactly two arguments.

### 1.4 Special Atoms: `this`, `super` (DD-01, DD-08)

- `this` → `{ type: "ThisExpression" }` (instead of Identifier)
- `super` → `{ type: "Super" }` (instead of Identifier)

These also work via colon syntax (1.2 handles it).

### 1.5 Remove `.` Form

The v0.0.1 `.` form for member access is removed. Replace with colon
syntax (static) and `get` (computed). Update existing tests.

### 1.6 Tests

- `test/forms/camel-case.test.js` — full edge case table
- `test/forms/colon-syntax.test.js` — splitting, chaining, this, super
- `test/forms/get.test.js` — computed access

## Phase 2 — Core Forms

**Dependencies**: Phase 1 (colon syntax, camelCase)
**Estimated effort**: ~3–4 days
**Items are independent** — can be built in any order within this phase.

### 2.1 `function` Declarations (DD-02)

`(function name (params) body...)` → `FunctionDeclaration`

- Name is **required** (no name → compile error, use `lambda`)
- Params is always a list
- Body: multiple statements, wrapped in `BlockStatement`

### 2.2 `async` Wrapper (DD-03)

`(async (function-form ...))` → sets `async: true` flag on the
enclosed function node.

- Confirm child is `function`, `lambda`, or `=>`
- Otherwise compile-time error

### 2.3 `await` (DD-03)

`(await expr)` → `AwaitExpression`. Unary. No context validation
(JS engine handles scoping).

### 2.4 `import` — Unified Form (DD-04)

Module path is ALWAYS the first argument (a string).

```
(import "mod")                         → side-effect import
(import "mod" name)                    → default import
(import "mod" (a b))                   → named imports
(import "mod" (a (alias b local)))     → named with rename
(import "mod" name (a b))              → default + named
```

ESTree: `ImportDeclaration`, `ImportSpecifier`, `ImportDefaultSpecifier`

**Banned**: namespace imports (`import *`). No `ImportNamespaceSpecifier`.

### 2.5 `export` — Unified Form (DD-04)

```
(export (const x 42))                  → export declaration
(export (function foo () ...))         → export function decl
(export (class Foo () ...))            → export class decl
(export default expr)                  → export default
(export (names a b))                   → export existing bindings
(export (names (alias a ext)))         → export with rename
(export "mod" (names a b))             → re-export named
```

ESTree: `ExportNamedDeclaration`, `ExportDefaultDeclaration`, `ExportSpecifier`

**Banned**: `export *` (re-export all). No `ExportAllDeclaration`.

### 2.6 `alias` Sub-form (DD-04, DD-06)

`(alias original local)` — used inside import/export specifier lists
AND inside destructuring patterns (Phase 4). Two arguments for rename,
three arguments for rename + default.

### 2.7 `dynamic-import` (DD-04)

`(dynamic-import expr)` → `ImportExpression`. Separate from `import`.

### 2.8 `throw` (Gap Analysis)

`(throw expr)` → `ThrowStatement`. Unary.

### 2.9 `try` / `catch` / `finally` (Gap Analysis)

```lisp
(try
  body...
  (catch e body...)
  (finally body...))
```

→ `TryStatement`, `CatchClause`

Catch and finally clauses recognized by head atom within try body.
Both optional, but at least one required.

### 2.10 `while` (Gap Analysis)

`(while test body...)` → `WhileStatement`

### 2.11 `do-while` (DD-08)

`(do-while test body...)` → `DoWhileStatement`

**Note**: Test comes FIRST (internal consistency), unlike JS syntax.

### 2.12 `for` (Gap Analysis)

`(for init test update body...)` → `ForStatement`

Init, test, update can be `()` for null/empty.

### 2.13 `for-of` (Gap Analysis)

`(for-of binding iterable body...)` → `ForOfStatement`

Left side is `VariableDeclaration` (kind: `const`). Binding can be
a plain atom or a destructuring pattern (Phase 4).

### 2.14 `for-in` (Gap Analysis)

`(for-in binding object body...)` → `ForInStatement`

Same structure as `for-of`.

### 2.15 `break` / `continue` (Gap Analysis)

```
(break)           → BreakStatement (no label)
(break my-label)  → BreakStatement (with label, camelCase applied)
(continue)        → ContinueStatement
(continue label)  → ContinueStatement (with label)
```

### 2.16 `switch` / `case` (Gap Analysis)

```lisp
(switch expr
  ("run" (do-run) (break))
  ("walk" (do-walk) (break))
  (default (do-nothing)))
```

→ `SwitchStatement`, `SwitchCase`

Each case is a list: first element is test (or `default` atom),
rest are body statements.

### 2.17 `?` Ternary (DD-08)

`(? test consequent alternate)` → `ConditionalExpression`

Three required arguments. NOT `?:` (colon would trigger splitting).

### 2.18 `++` / `--` Prefix Only (DD-08)

`(++ x)` → `UpdateExpression` (prefix: true, operator: `++`)
`(-- x)` → `UpdateExpression` (prefix: true, operator: `--`)

Postfix deferred to v0.2.0.

### 2.19 `**` Exponentiation (DD-08)

Add `**` to binary operator table → `BinaryExpression`.

### 2.20 All Compound Assignments (DD-08)

Register each operator. All take exactly 2 args → `AssignmentExpression`.

```
+=  -=  *=  /=  %=  **=
<<=  >>=  >>>=
&=  |=  ^=
&&=  ||=  ??=
```

### 2.21 `label` (DD-08)

`(label name body)` → `LabeledStatement`. Name gets camelCase.

### 2.22 `debugger` (DD-08)

`(debugger)` → `DebuggerStatement`. No arguments.

### 2.23 `seq` (DD-08)

`(seq expr1 expr2 ...)` → `SequenceExpression`. Two or more args.

### 2.24 `regex` (DD-08)

`(regex pattern)` or `(regex pattern flags)` → `Literal` with
`regex: { pattern, flags }`. Both args are strings.

### 2.25 Tests

One test file per form. Each test compiles a lykn snippet and asserts
the JS output string matches expected output.

## Phase 3 — Expressions

**Dependencies**: Phase 1. Independent of Phase 2.
**Estimated effort**: ~1 day

### 3.1 `template` Form (DD-05)

`(template args...)` → `TemplateLiteral`

- String args → `TemplateElement` (segments)
- Non-string args → entries in `expressions` array
- Compiler infers empty `TemplateElement` at boundaries:
  - If first arg is non-string → prepend empty element
  - If two adjacent args are non-string → insert empty between
  - If last arg is non-string → append empty element (tail: true)

### 3.2 `tag` Form (DD-05)

`(tag fn (template ...))` → `TaggedTemplateExpression`

Two args: tag expression + template form. Error if second arg is
not a `template` form.

### 3.3 `spread` in Expressions (DD-06)

`(spread expr)` → `SpreadElement`

Works inside `array`, `object`, and function call arguments.

### 3.4 `default` in Function Params (DD-06)

`(default name value)` in a function parameter list →
`AssignmentPattern`. Works in `function`, `lambda`, and `=>` params.

### 3.5 Object Construction — Grouped Pairs (DD-06 amendment)

**IMPORTANT**: The `object` construction syntax has changed from flat
alternating pairs to grouped pairs:

```lisp
;; OLD (remove): (object name "Duncan" age 42)
;; NEW:          (object (name "Duncan") (age 42))
```

Rules:
- `(key value)` sub-list → `Property` node (computed: false)
- Bare atom `name` → shorthand `Property` (key and value same Identifier)
- `(spread expr)` → `SpreadElement`
- `((computed key-expr) value)` → `Property` with computed: true
- `(object (name))` single-element sub-list → compile-time error

### 3.6 Tests

- `test/forms/template.test.js` — all segment inference cases
- `test/forms/tag.test.js`
- `test/forms/spread.test.js`
- `test/forms/default-params.test.js`
- `test/forms/object.test.js` — updated for grouped pair syntax

## Phase 4 — Patterns (Destructuring)

**Dependencies**: Phases 2 + 3 (`default`, `alias`, function params, `for-of`)
**Estimated effort**: ~2 days

### 4.1 Context Detection

The compiler must detect when `object` or `array` appears in pattern
position:
- Left side of `const`/`let`/`var` → pattern
- Left side of `=` → pattern
- Inside function parameter list → pattern
- Inside `for-of`/`for-in` binding → pattern

### 4.2 `object` in Pattern Position (DD-06)

`(const (object name age) person)` → `ObjectPattern`

- Bare atom `name` → shorthand binding
- `(alias key local)` → rename
- `(alias key local default-val)` → rename + default
- `(default name value)` → default value
- `(rest others)` → `RestElement` (must be last)

### 4.3 `array` in Pattern Position (DD-06)

`(const (array first second) arr)` → `ArrayPattern`

- Bare atom → binding
- `_` → skip (null in elements array)
- `(default name value)` → default
- `(rest tail)` → `RestElement` (must be last)

### 4.4 Nested Patterns (DD-06)

The target of `alias` can itself be a pattern:

```lisp
(const (object (alias data (array first second))) response)
```
→ `const { data: [first, second] } = response`

### 4.5 Destructuring in Assignment (DD-06)

When `=` has `object` or `array` on the left:

```lisp
(= (object a b) obj)
```
→ `AssignmentExpression` with `ObjectPattern` as left

### 4.6 Destructuring in Function Params (DD-06)

Patterns in parameter lists:

```lisp
(=> ((object name age)) (console:log name age))
(function process (req (object data (default status 200))) ...)
```

### 4.7 `rest` in Patterns (DD-06)

`(rest name)` → `RestElement`. Must be last element. Works in both
`object` and `array` patterns.

### 4.8 Tests

- `test/forms/destructuring-object.test.js`
- `test/forms/destructuring-array.test.js`
- `test/forms/destructuring-nested.test.js`
- `test/forms/destructuring-params.test.js`
- `test/forms/destructuring-assignment.test.js`

## Phase 5 — Classes

**Dependencies**: Everything (colon syntax, function forms, async,
destructuring, spread/rest, default, field)
**Estimated effort**: ~2–3 days

### 5.1 `class` Declaration (DD-07)

`(class Name (SuperClass) body...)` → `ClassDeclaration`

- Name is required
- Superclass is a list: `()` = no extends, `(Animal)` = extends Animal
- Body elements are methods, fields, or accessors

### 5.2 `class-expr` (DD-07)

`(class-expr (SuperClass) body...)` → `ClassExpression` (anonymous)

### 5.3 Method Definitions (DD-07)

Bare lists in class body: `(method-name (params) body...)`

No `method` keyword. The compiler knows it's in a class body.

### 5.4 `constructor` (DD-07)

`constructor` is a regular method name. Compiler sets
`kind: "constructor"` on the `MethodDefinition`.

### 5.5 `get` / `set` Keyword Markers (DD-07)

Inside class body ONLY:

```lisp
(get area () (return (* Math:PI (** this:radius 2))))
(set radius (r) (= this:radius r))
```

Compiler sets `kind: "get"` or `kind: "set"`.

**Note**: `get` as a keyword marker inside class bodies does NOT
conflict with the `get` form for computed access. Class body top-level
is a separate context from expression compilation.

**Restriction**: A method literally named `get` or `set` inside a
class body is shadowed by the accessor keyword.

### 5.6 `static` Wrapper (DD-07)

`(static (...))` wraps any class body member to make it static.
Composes with `async`: `(static (async (method-name (params) ...)))`.

Sets `static: true` on `MethodDefinition` or `PropertyDefinition`.

### 5.7 `field` for Class Fields (DD-07)

`(field name value)` or `(field name)` → `PropertyDefinition`

Explicit marker distinguishes fields from methods.

### 5.8 Private Fields/Methods via `-` Prefix (DD-07)

Inside a class body, leading `-` on a name applies:
1. DD-01's underscore conversion: `-balance` → `_balance`
2. Then prepends `#`: → `#_balance`

References use `this:-balance` which produces
`MemberExpression(ThisExpression, PrivateIdentifier("#_balance"))`.

### 5.9 `async` Methods (DD-03, DD-07)

`(async (method-name (params) body...))` inside class body.
The `async` wrapper composes with methods and `static`.

### 5.10 Tests

- `test/forms/class.test.js` — basic class, extends
- `test/forms/class-methods.test.js` — constructor, get/set, static
- `test/forms/class-fields.test.js` — field, private fields
- `test/forms/class-async.test.js` — async methods
- `test/forms/class-expr.test.js`

## Phase 6 — Browser Shim

**Dependencies**: Phases 1–5 (compiler must be feature-complete)
**Estimated effort**: ~1 day

### 6.1 Bundle Compiler

Bundle `reader.js` + `compiler.js` + `astring` as a single IIFE file
using esbuild:

```sh
esbuild src/lykn-browser.js --bundle --format=iife \
  --global-name=lykn --outfile=dist/lykn-browser.js --minify
```

### 6.2 `<script type="text/lykn">` Scanner

On `DOMContentLoaded`, query all `script[type="text/lykn"]` elements.
Process in document order.

### 6.3 Compile-then-Eval

For each script tag:
- Inline: compile `el.textContent` to JS, execute with `(0, eval)(js)`
- `src` attribute: fetch the file, compile, eval

Indirect eval ensures global scope execution.

### 6.4 `window.lykn` API

Expose for programmatic use:
- `window.lykn.compile(source)` → JS string
- `window.lykn.run(source)` → compile + eval
- `window.lykn.load(url)` → fetch + compile + eval

### 6.5 Error Handling

Errors to `console.error` with `[lykn]` prefix and script label.

### 6.6 Tests

- Manual browser test: HTML page with inline `<script type="text/lykn">`
- Manual browser test: HTML page with `<script type="text/lykn" src="app.lykn">`

## Integration Tests

After all phases, these complete programs must compile and run:

### Test 1: HTTP Server

```lisp
(import "node:http" (create-server))

(class Handler ()
  (field -count 0)

  (async (-fetch-data (url)
    (const response (await (fetch url)))
    (return (await (response:json)))))

  (async (handle (req res)
    (++ this:-count)
    (const data (await (this:-fetch-data
      "https://api.example.com/data")))
    (const (object name (default age 0)) data)
    (const body (template "Hello, " name "! Age: " age))
    (res:write-head 200 (object (content-type "text/plain")))
    (res:end body))))

(const handler (new Handler))

(const server (create-server (=> (req res)
  (handler:handle req res))))

(server:listen 3000 (=> ()
  (console:log "Server running on port 3000")))
```

### Test 2: CLI Tool

```lisp
(import "node:fs" (read-file-sync))
(import "node:path" (resolve))

(function process-file (path)
  (try
    (const content (read-file-sync (resolve path) "utf-8"))
    (const lines (content:split "\n"))
    (for-of line lines
      (if (> (line:trim):length 0)
        (console:log (template "> " line))))
    (catch e
      (console:error (template "Error: " e:message)))))

(const args (process:argv:slice 2))
(if (=== args:length 0)
  (console:log "Usage: node cli.js <file>")
  (process-file (get args 0)))
```

### Test 3: Utility Module

```lisp
(export (function clamp (val min-val max-val)
  (return (Math:min (Math:max val min-val) max-val))))

(export (function map-range (val in-min in-max out-min out-max)
  (const ratio (/ (- val in-min) (- in-max in-min)))
  (return (+ out-min (* ratio (- out-max out-min))))))

(export (const PI Math:PI))

(export default (object
  (clamp clamp)
  (map-range map-range)
  (PI PI)))
```

## What's OUT (v0.2.0+)

| Feature | Why deferred |
|---------|-------------|
| Generators / `yield` | Complex, needs iterator protocol |
| `for await...of` | Depends on generators |
| Optional chaining `?.` | Needs design for `?.` syntax |
| Postfix `++`/`--` | Naming unresolved, rarely needed |
| Named function expressions | Rare use case |
| Named class expressions | Rare use case |
| Computed method names | Interacts with generators |
| Static blocks | Rarely needed |
| MetaProperty | Niche |
| BigInt literals | Niche, needs reader changes |
| Macro system | Major feature, own design phase |

## Reference: Decision Docs

| DD | Topic | Key decisions |
|----|-------|---------------|
| 01 | Colon syntax + camelCase | Compile-time splitting, `.` removed, `get` for computed, leading `:` reserved |
| 02 | Function forms | `function` (decl), `lambda` (expr), `=>` (arrow) |
| 03 | Async/await | `async` wrapper, `await` unary, top-level await free |
| 04 | Modules | Unified `import`/`export`, `alias` for rename, `dynamic-import`, namespace/`export *` banned |
| 05 | Template literals | `template` with type dispatch, compiler infers empty segments, `tag` for tagged |
| 06 | Destructuring | `object`/`array` as patterns, `alias` reused, `default`, `rest`, `_` skip, `spread`. **Amended**: grouped pairs for construction |
| 07 | Classes | `class`/`class-expr`, bare methods, `get`/`set` markers, `static` wrapper, `field`, private via `-` prefix |
| 08 | Misc | `?` ternary, prefix `++`/`--`, all compound assignments, `**`, `debugger`, `label`, `do-while` test-first, `seq`, `regex` |
| 09 | Scope | 6 phases, ~12–15 days, browser shim included, reader unchanged |
