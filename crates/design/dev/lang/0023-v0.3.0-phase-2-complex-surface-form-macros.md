## Step 2: `type` Macro

```lisp
(type Option (Some :any value) None)
```

Expands to multiple top-level forms. This is the first macro that returns multiple forms â€” the macro should return an array (the expander's `expandExpr` already handles arrays via the splice path).

Wait â€” actually, macros return a single AST node. For multiple forms, wrap in `(block ...)`:

```
(block
  (function Some (value)
    (if (check...) (throw ...))
    (return (object (tag "Some") (value value))))
  (const None (object (tag "None"))))
```

### Parsing

- First arg: type name (atom) â€” not used in expansion but documents the type
- Remaining args: each is either:
  - A list `(ConstructorName :type field1 :type field2 ...)` â€” constructor with fields
  - An atom `ConstructorName` â€” zero-field constructor

### Constructor with fields expansion

```
(function Some (value)
  <type-check for each typed field>
  (return (object (tag "Some") (value value))))
```

Parse fields: walk pairwise (keyword, name). Build param list from names only. Build type checks. Build return object with `(tag "ConstructorName")` plus each `(fieldName fieldName)`.

### Zero-field constructor expansion

```
(const None (object (tag "None")))
```

## Step 3: `func` Macro â€” Single Clause

```lisp
(func add
  :args (:number a :number b)
  :returns :number
  :pre (> a 0)
  :post (> ~ 0)
  :body (+ a b))
```

### Detection: single vs multi-clause

- If the element after the function name is a keyword (`:args`, `:body`, etc.) â†’ single clause
- If the element after the function name is a list whose first element is a keyword â†’ multi-clause (each list is a clause)

### Single clause parsing

Walk args after name, collecting by keyword: `:args`, `:returns`, `:pre`, `:post`, `:body`.

### Zero-arg shorthand

No keywords present â€” form after name is directly the body.

```lisp
(func make-timestamp (Date:now))
```

â†’ `(function make-timestamp () (return (Date:now)))`

### Single clause expansion

```
(function funcName (param1 param2 ...)
  <type checks for each typed param>
  <pre-condition check>
  <if :post present>
    (const _result <body>)
    <post-condition check with ~ replaced by _result>
    (return _result)
  <else if :returns present>
    (return <body>)
  <else>
    <body>)
```

### Type check for params

Parse `:args` pairwise: `(:type name :type name ...)`. For each typed param (not `:any`), emit `typeCheck(name, type, funcName)`.

### Pre-condition

```
(if (! <pre-expr>) (throw (new Error "funcName: pre-condition failed: <source-sexpr> â€” caller blame")))
```

Use `formatSExpr` to serialize the pre-expression AST back to s-expression string for the error message.

### Post-condition

Replace `~` in the post expression with the gensym result variable. Walk the AST recursively replacing any atom with value `"~"` with the result gensym.

```
(const _result <body>)
(if (! <post-expr-with-tilde-replaced>) (throw (new Error "funcName: post-condition failed: <source-sexpr> â€” callee blame")))
(return _result)
```

### Returns type check

If `:returns` is present and not `:void` or `:any`, add type check on `_result` before returning.

## Step 4: `func` Macro â€” Multi-Clause

```lisp
(func greet
  (:args (:string name)
   :returns :string
   :body (+ "Hello, " name))
  (:args (:string greeting :string name)
   :returns :string
   :body (+ greeting ", " name)))
```

### Expansion

```
(function greet ((rest args))
  <if clause 1 matches>
    <clause 1 body with type checks and contracts>
  <else if clause 2 matches>
    <clause 2 body with type checks and contracts>
  ...
  (throw (new TypeError "greet: no matching clause for arguments")))
```

### Dispatch logic per clause

Each clause condition checks:

1. `(=== (get args :length) N)` â€” arity check (using `args.length` via `get` + atom `length`)

Wait â€” `args` is a rest param here, so it's a JS array. To get length: `args:length` via colon syntax.

Actually, `args` from `(rest args)` is a spread array. `(=== args:length N)` works via colon syntax.

For typed params, add type checks: `(&& (=== args:length 2) (=== (typeof (get args 0)) "string") ...)`

1. Bind params from args: `(const name (get args 0))`

### Ordering

Sort clauses: longer arity first, then more typed before less typed within same arity.

### Full expansion shape

```
(function greet ((rest args))
  (if (&& (=== args:length 1) (=== (typeof (get args 0)) "string"))
    (block
      (const name (get args 0))
      <type checks, pre, body, post>
      (return ...)))
  (if (&& (=== args:length 2) ...)
    (block
      (const greeting (get args 0))
      (const name (get args 1))
      ...
      (return ...)))
  (throw (new TypeError "greet: no matching clause for arguments")))
```

## Step 5: `match` Macro

```lisp
(match expr
  (pattern1 body1)
  (pattern2 :when guard body2)
  (_ default-body))
```

### Always IIFE

```
((=> ()
  (const _target <expr>)
  (if <pattern1-check> (block <bindings1> (return <body1>)))
  (if <pattern2-check> (block <bindings2> (return <body2>)))
  ...
  (throw (new Error "match: no matching pattern"))))
```

If the last clause is `_` (wildcard), no throw needed â€” the wildcard is the default.

### Pattern compilation â€” each pattern produces `{ check, bindings }`

**Literal patterns** (number, string, boolean, keyword, null, undefined):

- Check: `(=== _target <literal>)`
- Bindings: none

**Wildcard `_`**:

- Check: none (always matches)
- Bindings: none

**Symbol binding** (lowercase):

- Check: none (always matches, like wildcard but binds)
- Bindings: `(const <name> _target)`

**ADT constructor pattern** `(ConstructorName field1 field2 ...)`:

- Check: `(=== _target:tag "ConstructorName")`
- Bindings: for each field in pattern, `(const <field> _target:<fieldName>)`
- Field names come from the constructor definition â€” BUT the macro doesn't have access to type metadata at expansion time. Since fields in ADTs are named (not positional), the pattern bindings use the same names as the constructor fields. The macro needs a way to know field names.

**Problem**: The `match` macro doesn't know the field layout of ADT constructors at macro expansion time. The type metadata isn't available.

**Solution**: Use positional convention for match patterns. `(Some v)` means "match tag `Some`, bind the first field value to `v`". Since constructors have named fields, we need a mapping. BUT â€” we don't have it at macro time.

**Practical approach**: The match pattern `(Some v)` binds `v` to `_target.value` because `Some` was defined as `(Some :any value)` â€” the field is named `value`. But the macro can't know this.

**Simplification for JS path**: Use a convention where ADT pattern bindings are positional and the match macro generates property access by field index using `Object.values()` or `Object.keys()`.

Actually â€” looking at DD-17 more carefully, the examples show:

- `(Some v)` â†’ `const v = target.value` (field name from type def)
- `(Rect w h)` â†’ `const w = target.width; const h = target.height`

The field names must be available. Options:

1. **Type registry**: `type` macro registers constructor field names in a shared registry that `match` reads
2. **Use `Object.entries`**: Extract fields at runtime (fragile, order-dependent)
3. **Convention**: Pattern bindings match field names directly â€” `(Some value)` not `(Some v)`

Option 1 is cleanest. The `type` macro stores field metadata in a Map that `match` can query.

### Type registry (add to surface.js)

```javascript
const typeRegistry = new Map(); // "ConstructorName" â†’ ["field1", "field2", ...]
```

`type` macro populates it. `match` macro reads from it.

**ADT constructor pattern** `(ConstructorName binding1 binding2 ...)`:

- Look up field names: `typeRegistry.get("ConstructorName")` â†’ `["value"]`
- Check: `(=== _target:tag "ConstructorName")`
- Bindings: `(const binding1 _target:field1)`, `(const binding2 _target:field2)`, etc.
- The bindings use the user's chosen names, but access the correct field properties.

**Zero-field constructor pattern** `ConstructorName` (bare atom, PascalCase):

- Check: `(=== _target:tag "ConstructorName")`
- Bindings: none

**Structural obj pattern** `(obj :key1 binding1 :key2 binding2 ...)`:

- Check: `(&& (=== (typeof _target) "object") (!== _target null) (in "key1" _target) (in "key2" _target))`
- Bindings: `(const binding1 _target:key1)`, etc.
- For literal values in the pattern: add `(=== _target:key1 <literal>)` to check instead of binding

**Nested patterns**: Recursively compile inner patterns, generating intermediate gensym bindings and chaining checks.

**Guards** `:when`: Add guard expression with `&&` to the check.

## Step 6: `some->` / `some->>` Macros

```lisp
(some-> user (get :address) (get :city))
```

### Expansion (always IIFE)

```
((=> ()
  (const _t0 user)
  (if (== _t0 null) (return _t0))
  (const _t1 (get _t0 :address))    ;; thread-first: _t0 as first arg
  (if (== _t1 null) (return _t1))
  (return (get _t1 :city))))
```

Use `== null` (loose equality) for nil checks â€” this is the one `==` exception per DD-15/DD-18.

For each step:

- If atom: `(step _prev)` â€” bare function call
- If list: insert `_prev` as first arg (some->) or last arg (some->>)

## Step 7: `if-let` / `when-let` Macros

```lisp
(if-let ((Some v) expr) then-body else-body)
(when-let ((Some v) expr) body...)
```

### Parsing

First arg is a binding pair (a list of 2 elements): pattern and expression.

### Pattern detection

- If pattern is a list starting with PascalCase atom â†’ ADT constructor pattern
- If pattern is a list starting with `obj` â†’ structural pattern
- If pattern is a bare lowercase atom â†’ simple nil binding

### Always IIFE

```
((=> ()
  (const _t <expr>)
  (if <pattern-check>
    (block <bindings> (return <then-body>))
    (block (return <else-body>)))))
```

For `when-let`: same but no else block.

### Simple binding `(if-let (user (find-user id)) then else)`

```
((=> ()
  (const _t (find-user id))
  (if (!= _t null)
    (block (const user _t) (return <then>))
    (block (return <else>)))))
```

Note: `(!= _t null)` is `_t != null` which is truthy when not null/undefined.

Wait â€” `!=` is in the binaryOps. `(== _t null)` checks nil. `(!= _t null)` checks non-nil. Both use loose equality.

## Step 8: `fn` / `lambda` Macros

```lisp
(fn (:number x :number y) (+ x y))
(lambda (:any a :any b) (+ a b))
(fn () (Date:now))
```

### Expansion

Parse params pairwise `:type name` (same as func `:args`). Emit arrow function with type checks:

```
(=> (x y)
  <type checks>
  <body expressions>)
```

Multi-expression body:

```
(fn (:number x :string y)
  (console:log x)
  (+ x (parse-int y)))
```

â†’

```
(=> (x y)
  <type checks>
  (console:log x)
  (+ x (parse-int y)))
```

The `=>` kernel form already handles multi-expression bodies as block statements.

`lambda` is just an alias â€” register both pointing to the same function.
