# v0.3.0 Phase 6 — Macro Expander (Deno Subprocess, Three-Pass Pipeline)

## Context

Phases 3-5 built the `lykn-lang` Rust crate with reader, AST, classifier, diagnostics, analysis, and emitter. Phase 6 adds user-defined macro expansion — the `macro` and `import-macros` forms. Built-in surface forms (bind, func, match, etc.) are already handled by the classifier as static transforms. This module handles user-defined macros only, evaluating them via a Deno subprocess.

## New Files

```
crates/lykn-lang/src/
  expander/
    mod.rs              — public expand API, three-pass orchestration
    deno.rs             — Deno subprocess invocation protocol

    cache.rs            — Module cache (path + mtime)
```

## Architecture

### Pipeline Flow

```
Source → Reader → SExpr tree
         ↓
    [Expander] Pass 0: import-macros → load + compile macro modules
         ↓
    [Expander] Pass 1: compile file-local macro defs (iterative fixed-point)
         ↓
    [Expander] Pass 2: expand all macro invocations (top-down, fixed-point)
         ↓
    Classifier → Surface AST (re-classify expanded forms)
         ↓
    Analysis → Diagnostics
         ↓
    Emitter → Kernel JSON
```

### Deno Subprocess Protocol

The Rust expander communicates with a Deno subprocess to evaluate macro bodies.

**Input** (JSON to Deno stdin):

```json
{
  "action": "eval",
  "macroBody": "return (function(test, ...body) { return $array($sym('if'), test, $concat($array($sym('block')), body)); });",
  "args": [
    {"type": "atom", "value": "x"},
    {"type": "number", "value": 42}
  ]
}
```

**Deno evaluator script** (embedded in Rust, written to temp file or passed via `deno eval`):

```javascript
// Reads JSON from stdin, evaluates macro, writes result to stdout
const input = JSON.parse(await Deno.readAll(Deno.stdin));
// ... construct env API, call macro, serialize result ...
```

**Output** (JSON from Deno stdout):

```json
{"type": "list", "values": [{"type": "atom", "value": "if"}, ...]}
```

**Implementation**: Launch `deno eval` with the evaluator script. Pipe JSON in via stdin, read JSON from stdout. Keep a single subprocess alive for the compilation run (avoid per-macro spawn overhead).


```rust
pub fn sexpr_to_protocol_json(expr: &SExpr) -> serde_json::Value;
pub fn protocol_json_to_sexpr(val: &serde_json::Value) -> Result<SExpr, LyknError>;
```

### `expander/deno.rs` — Subprocess Management

```rust
pub struct DenoSubprocess {
    child: std::process::Child,
    stdin: BufWriter<ChildStdin>,
    stdout: BufReader<ChildStdout>,
}

impl DenoSubprocess {
    pub fn spawn() -> Result<Self, LyknError>;
    pub fn eval_macro(&mut self, js_body: &str, args: &[SExpr]) -> Result<SExpr, LyknError>;
    pub fn compile_macro(&mut self, js_body: &str) -> Result<String, LyknError>;
}
```

Error handling:

- Deno not installed → `LyknError` with "user-defined macros require Deno — install from deno.land"
- Subprocess crash → capture stderr, emit diagnostic
- Timeout → configurable, default 10s per macro evaluation

### `expander/compile.rs` — Macro Body Compilation

Convert lykn macro body s-expressions to JS source code:

```rust
pub fn compile_macro_body(
    params: &SExpr,    // parameter pattern
    body: &[SExpr],    // body forms
) -> Result<String, LyknError>;
```

This replicates the JS `compileMacroBody()` logic:

1. Extract parameter names from pattern (handle `rest`, `default`, nested destructuring)
2. Process body forms — compile quasiquote templates to API calls
3. Wrap in lambda form
4. Compile lambda to JS using the kernel compiler (or a simplified JS codegen)

**Practical approach**: Rather than reimplementing the full JS compiler in Rust, serialize the macro definition as s-expressions and send to Deno for compilation. The Deno subprocess compiles the macro body using the existing JS `compileMacroBody()` and returns the compiled JS function source.

### `expander/quasiquote.rs` — Quasiquote Compilation

If we do compile locally (vs delegating to Deno):

```rust
pub fn compile_quasiquote(form: &SExpr, depth: usize) -> SExpr;
```

Transforms quasiquote templates to API call trees:

- Atoms → `($sym "name")`
- Numbers/strings → pass through
- Keywords → pass through
- Unquote → variable reference
- Unquote-splicing → spread into `$concat`
- Lists → `$array(...)` or `$concat(...)` if splicing present

### `expander/pass0.rs` — Import Macros

```rust
pub fn process_import_macros(
    forms: &[SExpr],
    file_path: &Path,
    deno: &mut DenoSubprocess,
    cache: &mut ModuleCache,
    compilation_stack: &[PathBuf],
) -> Result<(Vec<SExpr>, MacroEnv), LyknError>;
```

- Find `import-macros` forms
- Resolve paths (relative to current file)
- Check cache (path + mtime)
- Check for circular dependencies (compilation_stack)
- Load and compile macro module recursively
- Register imported macros
- Return remaining forms (import-macros erased)

### `expander/pass1.rs` — Local Macro Compilation

```rust
pub fn compile_local_macros(
    forms: &[SExpr],
    deno: &mut DenoSubprocess,
    env: &mut MacroEnv,
) -> Result<Vec<SExpr>, LyknError>;
```

- Separate macro definitions from other forms
- Iterative fixed-point: compile macros with no dependencies first
- Detect circular dependencies (no progress → error)
- Return non-macro forms

### `expander/pass2.rs` — Expansion Walk

```rust
pub fn expand_all(
    forms: &[SExpr],
    deno: &mut DenoSubprocess,
    env: &MacroEnv,
) -> Result<Vec<SExpr>, LyknError>;

fn expand_expr(
    form: &SExpr,
    deno: &mut DenoSubprocess,
    env: &MacroEnv,
) -> Result<SExpr, LyknError>;
```

- Recursive top-down walk
- Dispatch table: quote → none, sugar forms → desugar, macro → expand
- Fixed-point per-node: re-expand until not a macro call
- Expansion limit: 1000 per node
- Desugar: cons, list, car, cdr, cadr, cddr, as

### `expander/cache.rs` — Module Cache

```rust
pub struct ModuleCache {

struct CacheEntry {

    macros: HashMap<String, CompiledMacro>,
}
```

```rust
pub type MacroEnv = HashMap<String, CompiledMacro>;
```

## Awareness Items

| Item | Impact |
|------|--------|
| **Deno is a required runtime dependency** | Macro expansion fails without deno. Clear error message provided. |
| **Subprocess overhead** | Single long-lived process per compilation run to amortize spawn cost. |
| **Macro compilation delegated to Deno** | Rather than reimplementing JS codegen in Rust, we send macro bodies to Deno for compilation using the existing JS `compileMacroBody()`. This ensures identical behavior. |
| **Re-classification required** | Expanded forms (raw SExpr from Deno) must be re-classified through the classifier to produce typed surface AST nodes. |
| **No persistent cross-run cache** | Cache is per-compilation-run only. Future enhancement. |
