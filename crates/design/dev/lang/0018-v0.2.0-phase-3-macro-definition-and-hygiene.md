# v0.2.0 Phase 3 — Macro Definition and Hygiene

## Context

v0.2.0 Phases 1 (reader extensions) and 2 (expander core) are complete. Phase 3 adds the macro system: `macro` form processing, compile-time evaluation via `new Function()`, enforced gensym hygiene, auto-gensym `#gen`, and `(sym)` escape hatch. This transforms lykn from a compile-to-JS language into a proper Lisp with macros.

## Implementation Order

1. **`compileQuasiquote(form)`** — compile QQ templates to API-call s-expressions (NOT resolve them). Different from Phase 2's `expandQuasiquote` which resolves.
2. **`compileMacroBody(params, bodyForms)`** — compile body to JS string using compiler
3. **`registerMacro(name, params, body)`** — `new Function()` wrapping + env storage
4. **Pass 1** — iterative fixed-point macro scanning and compilation
5. **Pass 2 update** — macro calls trigger fixed-point per-node expansion (limit 1000)
6. **`#gen` auto-gensym** — scanner walks QQ templates, allocates gensyms per prefix
7. **Enforced gensym** — detect bare symbols in binding positions inside QQ
8. **`(sym)` handling** — bypass gensym check, compile to `sym()` call
9. **Error reporting** — catch macro throws, format with call-site location

## Key Architecture

### `compileQuasiquote` vs `expandQuasiquote`

Phase 2's `expandQuasiquote` RESOLVES quasiquotes at expansion time. Phase 3 needs `compileQuasiquote` that COMPILES them to API calls:

```
`(if ,test (do ,@body))
→ expandQuasiquote (Phase 2): produces (array (quote if) test (append (array (quote do)) body))
→ compileQuasiquote (Phase 3): produces s-expr that compiles to:
   array(sym("if"), test, concat(array(sym("do")), body))
```

### `new Function()` Sandbox

```javascript
const macroFn = new Function(
  "array", "sym", "gensym",
  "isArray", "isSymbol", "isNumber", "isString",
  "first", "rest", "concat", "nth", "length",
  "append", "quote",
  compiledBodyString  // "return function(test, ...body) { return array(sym('if'), test, ...); }"
);
```

### Two-Pass Pipeline

```
Pass 1: Scan for (macro ...) → compile + register (iterative fixed-point)
Pass 2: Expand all forms (existing walk + macro env populated)
```

Pass 1 iterative fixed-point handles macro ordering:

- Pending = all macros; loop: compile macros with no unresolved deps → progress
- No progress + pending remaining → circular dependency error

### Safety Limits

- Per-node expansion: 1000 iterations
- Pass 1 compilation: N passes (N = macro count)
- Duplicate macro names: hard error
