# v0.3.0 Phase 4 — Analysis Module

**Type Registry, Scope Tracker, Maranget's Algorithm**

## Context

Phase 3 built the `lykn-lang` crate with reader, AST, classifier, and diagnostics. Phase 4 adds the analysis module — the static analysis layer that sits between classification and emission in the DD-20 pipeline. This includes the type registry, scope tracker, Maranget's exhaustiveness/usefulness algorithm, match exhaustiveness checking, and func overlap detection.

## New Files

```
crates/lykn-lang/src/

```

## Module 1: Type Registry (`analysis/type_registry.rs`)

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct FieldDef {
    pub name: String,
    pub type_keyword: String,  // "number", "string", "any", etc.
}

#[derive(Debug, Clone, PartialEq)]
pub struct ConstructorDef {
    pub name: String,
    pub fields: Vec<FieldDef>,
    pub owning_type: String,
    pub span: Span,
}

#[derive(Debug, Clone, PartialEq)]
pub struct TypeDef {
    pub name: String,
    pub module_path: Option<String>,
    pub constructors: Vec<ConstructorDef>,
    pub is_blessed: bool,
    pub span: Span,
}

#[derive(Debug, Default)]
pub struct TypeRegistry {
    types: HashMap<String, TypeDef>,
    constructor_to_type: HashMap<String, String>,
}

impl TypeRegistry {
    pub fn register_type(&mut self, typedef: TypeDef) -> Result<(), Diagnostic>;
    pub fn lookup_type(&self, name: &str) -> Option<&TypeDef>;
    pub fn lookup_constructor(&self, name: &str) -> Option<&ConstructorDef>;
    pub fn owning_type_of(&self, constructor: &str) -> Option<&TypeDef>;
    pub fn constructors_of(&self, type_name: &str) -> Option<&[ConstructorDef]>;
}
```

### Population

Walk all `SurfaceForm::Type` nodes, extract constructors, validate:

- No duplicate constructor names across all types
- All fields have type annotations (already enforced by classifier)

### Blessed types

After population, mark types with `module_path == Some("lykn/core/option")` or `"lykn/core/result"` as blessed.

## Module 2: Prelude Injection (`analysis/prelude.rs`)

Before classification, inject prelude SExpr forms:

```rust
pub fn inject_prelude(forms: &mut Vec<SExpr>) {
    // Prepend: (import (Option Some None) "lykn/core/option")
    // Prepend: (import (Result Ok Err) "lykn/core/result")
}
```

These get classified as `KernelPassthrough` (import is a kernel form). The type registry recognizes them via module path when `Type` forms with matching names are registered.

Actually — prelude injection needs the type definitions themselves, not just imports. For v0.3.0 JS path, types are defined at use-site. For the Rust analysis path, we need the type definitions to be available in the registry.

**Practical approach for v0.3.0**: Register Option and Result as built-in blessed types in the type registry directly (hardcoded), without requiring actual import/type forms. The prelude import forms are still injected for the kernel emitter to produce the JS `import` statements.

```rust
pub fn register_prelude_types(registry: &mut TypeRegistry) {
    registry.register_type(TypeDef {
        name: "Option".into(),
        module_path: Some("lykn/core/option".into()),
        constructors: vec![
            ConstructorDef { name: "Some".into(), fields: vec![FieldDef { name: "value".into(), type_keyword: "any".into() }], owning_type: "Option".into(), span: Span::default() },
            ConstructorDef { name: "None".into(), fields: vec![], owning_type: "Option".into(), span: Span::default() },
        ],
        is_blessed: true,
        span: Span::default(),
    }).ok();
    // Same for Result with Ok/Err
}
```

## Module 3: Scope Tracker (`analysis/scope.rs`)

```rust
#[derive(Debug, Clone)]
pub struct Binding {
    pub name: String,
    pub span: Span,
    pub used: bool,
    pub exported: bool,
    pub is_constructor: bool,
    pub underscore_prefixed: bool,
}

#[derive(Debug)]
pub struct Scope {
    bindings: HashMap<String, Binding>,
    parent: Option<Box<Scope>>,
}

#[derive(Debug, Default)]
pub struct ScopeTracker {
    root: Scope,
    diagnostics: Vec<Diagnostic>,
}

impl ScopeTracker {
    pub fn enter_scope(&mut self);
    pub fn exit_scope(&mut self);
    pub fn introduce(&mut self, name: &str, span: Span, exported: bool, is_constructor: bool);
    pub fn reference(&mut self, name: &str, span: Span);
    pub fn collect_diagnostics(&mut self) -> Vec<Diagnostic>;
}
```

### Binding introduction rules (from DD-21 binding table)

| Form | Bindings | Scope |
|------|----------|-------|
| `Bind` | name | rest of block |
| `Func` | function name + params per clause | name: enclosing; params: body |
| `Fn`/`Lambda` | params | body |
| `Match` clause | pattern bindings | clause body |
| `IfLet` | pattern bindings | then-branch |
| `WhenLet` | pattern bindings | body |
| `Type` | constructor names | rest of block |
| `KernelPassthrough(const)` | name | rest of block |
| `KernelPassthrough(function)` | name + params | same as Func |
| `KernelPassthrough(import)` | imported names | module |

### Diagnostics produced

- **Shadowing**: warning when a binding shadows an outer binding
- **Unused**: warning when a binding is never referenced (exempt: `_` prefix, exported, constructor)
- **Undefined**: error for surface form references, warning for kernel passthrough references

## Module 4: Deconstructed Patterns (`analysis/pattern.rs`)

Internal pattern representation for Maranget's algorithm, separate from the surface `Pattern` AST.

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum DeconPattern {
    /// ADT constructor: name + sub-patterns for fields
    Constructor {
        type_name: String,
        ctor_name: String,
        fields: Vec<DeconPattern>,
    },
    /// Literal value (from finite or infinite type)
    Literal(LiteralKind),
    /// Wildcard — matches anything
    Wildcard,
    /// Structural obj pattern — open type, always needs wildcard
    Structural {
        keys: Vec<(String, DeconPattern)>,
    },
    /// Type keyword in func :args (treated as constructor from built-in type set)
    TypeKeyword(String),
}

#[derive(Debug, Clone, PartialEq)]
pub enum LiteralKind {
    Number(OrderedFloat),  // or i64/f64 with custom Eq
    String(String),
    Bool(bool),
    Null,
    Keyword(String),
}
```

### Conversion from surface Pattern

```rust
pub fn deconstruct_pattern(pat: &Pattern, registry: &TypeRegistry) -> Result<DeconPattern, Diagnostic>;
```

- `Pattern::Wildcard` → `DeconPattern::Wildcard`
- `Pattern::Binding` → `DeconPattern::Wildcard` (bindings match anything)
- `Pattern::Literal(SExpr::Number{..})` → `DeconPattern::Literal(Number(..))`
- `Pattern::Constructor{name, bindings}` → look up in registry, `DeconPattern::Constructor{..}` with recursively deconstructed sub-patterns
- `Pattern::Obj{pairs}` → `DeconPattern::Structural{..}`

## Module 5: Maranget's Algorithm (`analysis/maranget.rs`)

Core implementation following Maranget 2007.

```rust
/// A pattern matrix: rows are clauses, each row is a vector of deconstructed patterns.
pub type PatternMatrix = Vec<Vec<DeconPattern>>;

/// Check if a pattern vector is useful with respect to a matrix.
pub fn is_useful(matrix: &PatternMatrix, vector: &[DeconPattern], registry: &TypeRegistry) -> bool;

/// Check if a matrix is exhaustive (covers all possible inputs).
pub fn is_exhaustive(matrix: &PatternMatrix, col_count: usize, registry: &TypeRegistry) -> bool;

/// Find uncovered patterns (witness of non-exhaustiveness).
pub fn find_uncovered(matrix: &PatternMatrix, col_count: usize, registry: &TypeRegistry) -> Vec<Vec<DeconPattern>>;
```

### Key operations

**Specialization** (`specialize`): Given a constructor `c` with arity `a`, produce a new matrix where:

- Rows with head constructor `c`: expand to `a` new columns from the fields + remaining columns
- Rows with wildcard head: expand to `a` wildcards + remaining columns
- Rows with different constructor head: removed

**Default matrix** (`default`): Rows where the head is a wildcard (constructor rows removed), with the first column stripped.

**Algorithm**:

1. If matrix is empty: the vector IS useful (no row covers it)
2. If vector is empty: the vector is NOT useful (everything is covered)
3. Look at first column. Get the set of constructors `C` in that column.
4. If `C` is complete (all constructors of a type present): specialize by each constructor, recurse
5. If `C` is incomplete: check default matrix + specialize by each present constructor

### f64 equality

For literal number patterns, we need equality comparison. Use `f64::to_bits()` for Eq/Hash instead of adding a dependency.

## Module 6: Match Exhaustiveness (`analysis/match_check.rs`)

```rust
pub fn check_match(
    match_form: &SurfaceForm,  // SurfaceForm::Match
    registry: &TypeRegistry,
) -> Vec<Diagnostic>;
```

### Five inference rules

1. **ADT constructor patterns → infer type**: Lookup first constructor in registry → get type
2. **All constructors same type**: Verify all constructor patterns belong to same type, else error
3. **Literal-only → require `_`**: No type inference, check `_` present
4. **Structural `obj` → require `_`**: No type inference, check `_` present
5. **Mixed constructor + literal → type error**: Contradictory, compile error

### Process

1. Extract patterns from match clauses
2. Apply inference rules to determine type (if any)
3. Convert to `DeconPattern` via `deconstruct_pattern`
4. Build pattern matrix (1 column for simple match, N columns for nested)
5. Run exhaustiveness check
6. Run usefulness per clause for unreachability warnings
7. Guard handling: exclude guarded clauses from exhaustiveness matrix

### Enhanced messages for blessed types

```rust
fn format_missing_variant(type_name: &str, variant: &str, is_blessed: bool) -> String {
    if is_blessed && type_name == "Option" && variant == "None" {
        "this function can return None but you haven't handled the empty case".into()
    } else if is_blessed && type_name == "Result" && variant == "Err" {
        "this Result match doesn't handle the error case".into()
    } else {
        format!("non-exhaustive match — missing {variant}")
    }
}
```

## Module 7: Func Overlap Detection (`analysis/func_check.rs`)

```rust
pub fn check_func_overlap(
    func_form: &SurfaceForm,  // SurfaceForm::Func with multiple clauses
    registry: &TypeRegistry,
) -> Vec<Diagnostic>;
```

### Process

1. For each clause, extract `:args` type keywords as pattern row
2. Group clauses by arity
3. Within each arity group, build pattern matrix with type keywords as constructors
4. For each clause pair, check if either is useful w.r.t. the other → overlap if neither is useful
5. Report overlapping clause pairs with span info

### Type keyword mapping

`:number` → constructor `Number`, `:string` → `String`, `:any` → wildcard, etc.

### Orchestration (two phases per DD-20)

**Phase 1 — Collection**:

1. Register prelude types (Option, Result)
2. Walk all forms, register user-defined types
3. Initialize scope tracker with top-level imports

**Phase 2 — Analysis**:

1. For each form, run form handler (scope tracking + form-specific checks)
2. Match nodes: run exhaustiveness check
3. Func nodes with multiple clauses: run overlap detection
4. Collect scope diagnostics (unused, shadowing, undefined)

## Testing Strategy

Tests go in `#[cfg(test)]` modules within each file.

### Type Registry tests

- Register type, lookup constructor → correct owning type
- Duplicate constructor → error
- Blessed type identification

### Scope Tracker tests

- Introduce + reference → no warning
- Introduce without reference → unused warning
- `_` prefix → no unused warning
- Shadowing → warning
- Undefined reference → error

### Maranget tests

- Exhaustive: all constructors covered → `is_exhaustive` returns true
- Non-exhaustive: missing variant → returns false, `find_uncovered` identifies it
- Wildcard covers remainder
- Nested patterns: all combinations
- Literal patterns: open type, wildcard needed

### Match check tests

- All ADT variants covered → no diagnostic
- Missing variant → error naming it
- Guarded clause → excluded from coverage
- Mixed constructor types → error
- Mixed constructor + literal → error
- Literal-only without `_` → error
- Unreachable clause → warning
- Enhanced message for blessed Option/Result

### Func overlap tests

- Same arity same types → overlap error
- Different arity → no overlap
- `:any` overlapping typed → overlap error
- Different types same arity → no overlap

## Awareness Items

| Item | Impact |
|------|--------|
| **f64 equality for literal patterns** | Using `f64::to_bits()` for comparison avoids adding `ordered-float` dependency |
| **Prelude types are hardcoded** | Option/Result registered directly, not from actual `.lykn` files. Revisit when module system is implemented. |
| **Scope tracking for kernel forms is limited** | Only recognizes `const`, `function`, `import` heads in KernelPassthrough. Other kernel forms that introduce bindings need future expansion. |
| **No cross-module analysis** | Scope and exhaustiveness are per-module only. Cross-module type checking deferred. |
