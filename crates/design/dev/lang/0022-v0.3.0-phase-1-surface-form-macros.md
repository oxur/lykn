# Plan: v0.3.0 Phase 1 — Surface Form Macros

## Context

lykn/surface forms are syntactic sugar that expand to lykn/kernel forms. Phase 1 implements the foundation surface macros (`bind`, `obj`, `cell`/`express`/`swap!`/`reset!`, `->`, `->>`, `assoc`, `dissoc`, `conj`) in the JS macro expander. These macros are the reference implementation — the Rust compiler will produce identical output later.

**Prerequisite**: DD-15 specifies keywords (`:name`) as a reader-level type. The reader currently produces `{ type: 'atom', value: ':name' }` and the compiler throws "Leading colon syntax is reserved for future use". Keywords must be activated first since `obj`, `assoc`, `dissoc`, and `bind` (with type annotations) all depend on them.

## Step 2: Surface Macros (`src/surface.js` — new file)

Export `registerSurfaceMacros(macroEnv)`. Import `{ sym, array, gensym, isKeyword, isArray }` from `./expander.js`. Each macro is a plain JS function receiving AST args, returning AST.

### 2a. `bind`

```
(bind name value)        → (const name value)
(bind :type name value)  → (const name value)   // type annotation dropped for JS
```

If first arg is keyword, skip it. Emit `array(sym('const'), name, value)`.

### 2b. `obj`

```
(obj :name "Duncan" :age 42) → (object (name "Duncan") (age 42))
```

Walk args pairwise: keyword → `sym(keyword.value)` as pair key, next → pair value. Return `array(sym('object'), ...pairs)`. The compiler's `toCamelCase` handles kebab-to-camel on the atom when compiling.

### 2c. `cell`

```
(cell value) → (object (value value))
```

Return `array(sym('object'), array(sym('value'), arg))`.

### 2d. `express`

```
(express c) → c:value
```

Return `sym(arg.value + ':value')`. Compiler's colon syntax produces `c.value`.

### 2e. `swap!`

```
(swap! c f)       → (= c:value (f c:value))
(swap! c f a b)   → (= c:value (f c:value a b))
```

Build `array(sym('='), sym(c.value + ':value'), array(f, sym(c.value + ':value'), ...extraArgs))`.

### 2f. `reset!`

```
(reset! c v) → (= c:value v)
```

Build `array(sym('='), sym(c.value + ':value'), v)`.

### 2g. `->` (thread-first)

```
(-> x f g)           → (g (f x))
(-> x (f a) (g b))   → (g (f x a) b)
```

Reduce: for each step, if atom → `array(step, threaded)`. If list → insert threaded as second element: `array(first, threaded, ...rest)`.

### 2h. `->>` (thread-last)

```
(->> x (f a) (g b)) → (g a (f b x))
```

Same as `->` but threaded value goes last: `array(first, ...rest, threaded)`.

### 2i. `assoc`

```
(assoc obj :key val)       → (object (spread obj) (key val))
(assoc obj :a 1 :b 2)     → (object (spread obj) (a 1) (b 2))
```

First arg is object. Walk remaining args as keyword-value pairs. Build `array(sym('object'), array(sym('spread'), obj), ...pairs)`.

### 2j. `dissoc` — IIFE pattern

```
(dissoc obj :key) → ((=> () (const (object (alias key _g0) (rest _g1)) obj) _g1))
```

Expansion uses an immediately-invoked arrow function:

- For each key to remove: `(alias keyName gensym())` in the destructuring pattern
- `(rest gensym())` captures everything else
- The IIFE returns the rest variable

For multiple keys `(dissoc obj :a :b)`:

```
((=> ()
  (const (object (alias a _g0) (alias b _g1) (rest _g2)) obj)
  _g2))
```

### 2k. `conj`

```
(conj arr val) → (array (spread arr) val)
```

Build `array(sym('array'), array(sym('spread'), arr), val)`.

## Implementation Order

1. Keywords (reader → compiler → expander API) — prerequisite for everything
2. Surface macro registration scaffolding (`surface.js` + expander integration)
3. `bind` — simplest macro, validates registration pipeline
4. `obj` — keyword-dependent, validates keyword pipeline end-to-end
5. `cell`/`express`/`swap!`/`reset!` — related group
6. `->` / `->>` — independent
7. `assoc` / `conj` — straightforward
8. `dissoc` — most complex (IIFE + destructuring + gensym)
9. JSON fixtures for all macros
10. Integration test

## Unplanned Changes & Awareness Items

Changes introduced by this work that other parts of the system/design need to be aware of:

| Item | Impact | Affects |
|------|--------|---------|
| **Keyword node type added to reader** | New AST node `{ type: 'keyword', value }`. Any code that exhaustively switches on node types needs updating. | Rust reader (`crates/lykn-cli/src/reader.rs`), any AST walkers, formatters |
| **`isKeyword` added to macro API** | User-defined macros can now detect and process keyword args. This is a new API surface. | DD-13 documentation, macro authoring guide |
| **`dissoc` uses IIFE pattern** | `dissoc` expands to `((=> () ...))` — an immediately-invoked arrow function. This works but produces slightly verbose JS. Future phases may want a `do`-expression or block-expression form to simplify this. | DD-15 design, Rust surface compiler |
| **Keywords strip `:` at reader level** | Macro authors receive `{ type: 'keyword', value: 'name' }` not `':name'`. The `:` prefix is gone. | Macro authoring conventions, documentation |
| **Surface macros are "built-in" macros** | They occupy names in the macro namespace. User `(macro bind ...)` would conflict (duplicate error). May need a way to shadow or override. | Macro namespace design, error messages |
| **`express`/`swap!`/`reset!` assume atom args** | These macros use `arg.value + ':value'` which only works for simple atom arguments. Complex expressions (e.g., `(express (get cells 0))`) would need different handling. | Future cell design refinements |
| **Leading colon error removed from compiler** | The "reserved for future use" error for `:foo` atoms is removed. Bare `:` is still an error. | Compiler error handling |
